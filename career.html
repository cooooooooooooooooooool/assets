2016.04

~

2017.12

노후산단 Cloud FEMS 구축
  • (이슈) OPC, Modbus 등의 계측 데이터 수집 모듈이 계측 데이터의 태그 개수가 1,000 태그 이상 많은 경우 out of memory 로 java process 가 종료되는 현상 발생
  • (해결) java 실행시 heap size 및 파라미터 최적화와 500 태그 기준으로 데이터 수집 java 프로세스를 나누고 5분 주기로 이들 프로세스를 모니터링 하는 agent 프로그램을 개발하여 장애 발생시 즉각 대응하도록 조치
  • (이슈) 계측 데이터(Time Series Data)를 Hadoop 에 저장하기 위해서 지나치게 많은 인프라(최소 5개의 OS 에 Hadoop 설치 필요)가 필요하여 관리 비용 증가
  • (해결) 계측 데이터를 손쉽게 저장하고 인프라의 확장이 용이한 Cassandra + KairosDB 조합으로 변경 → 2개의 Linux OS 에 설치 및 관리
  • (이슈) 가장 최근에 수집한 계측 데이터의 빠른 처리를 위해 메모리 DB(Redis) 에 저장하여 액세스 하는데 Redis 서버가 1달 간격으로 프로세스가 종료되는 현상 발생
  • (해결) 에러 로그를 분석하여 Redis 설정값을 변경하여 적용하였지만 동일한 문제 발생. Redis 메모리 사용량을 최소화하고 4개의 Redis 노드와 3개의 Sentinel 노드로 구성하여 Master Redis 노드가 종료되더라도 나머지 3개의 Slave 노드 중 1개의 노드가 Master 노드로 전환되어 서비스가 지속되도록 인프라 구성을 변경하여 해결함

  • (해결) jdk 의 toolkit 을 이용하여 .pfx 인증서를 .jks 유형의 인증서로 변환하여 적용
  • (이슈) MariaDB, RabbitMQ 의 이중화 테스트를 위해 L4 스위치가 필요한 상황인데 L4 스위치를 섭외할 수 없어 이중화 테스트를 수행하지 못하는 상황 발생
  • (해결) HAProxy 와 KeepAlived 를 조합하여 2대의 서버에 나누어 설치하고 가상 IP 를 부여하여 L4 스위치와 동일한 작동을 하는 시스템을 구현하여 이중화 테스트 수행
  • (이슈) Workcenter 성능 테스트에서 응답 시간이 많이 소요되는 REST API 문제 발생
  • (해결) 알림 메시지와 같이 데이터가 많이 쌓이는 경우 정규화된 테이블 데이터로 인해 SQL 실행시 index full scan 발생. 데이터 생성 주기가 짧은 테이블은 반정규화를 수행하고 자주 발생하는 SQL 에 대해 인덱스를 생성하여 데이터 검색이 빠르게 처리되도록 해결
